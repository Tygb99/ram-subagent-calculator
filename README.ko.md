# AgentFit

[English](README.md) · [계산기 열기](https://tygb99.github.io/ram-subagent-calculator/)

AgentFit은 컴퓨터 RAM 안에서 코딩 서브에이전트를 몇 개까지 동시에 실행할 수 있는지 추정하는, 설치가 필요 없는 공개 웹 계산기입니다.

전체 RAM을 입력하고 작업 강도를 선택하면 세 값을 비교할 수 있습니다.

- **안전**: 계산된 최대치의 65%로, 장시간 작업에 적합한 보수적 수치입니다.
- **균형**: 최대치의 85%로, 기본 권장값입니다.
- **최대**: 메모리만 고려한 계산상 상한이며 갑작스러운 메모리 증가에 취약합니다.

## 계산 방식

운영체제와 다른 앱이 쓸 메모리를 먼저 남기고, 메인 에이전트 1개에 필요한 RAM을 제외한 뒤 남은 풀을 서브에이전트에 배분합니다.

```text
시스템 여유분   = clamp(전체 RAM × 25%, 3 GB, 16 GB)
서브에이전트 풀 = max(0, 전체 RAM − 시스템 여유분 − 메인 에이전트 RAM)
최대            = floor(서브에이전트 풀 ÷ 에이전트당 RAM)
안전            = floor(최대 × 65%)
균형            = floor(최대 × 85%)
```

계산된 최대값이 1이면 안전과 균형도 1로 유지합니다.

| 작업 강도 | 에이전트당 RAM | 적합한 작업 |
| --- | ---: | --- |
| 가벼움 | 1 GB | 코드 탐색, 작은 수정, 짧은 프롬프트 |
| 균형 | 2 GB | 일반 구현과 리뷰 |
| 무거움 | 4 GB | 빌드, 브라우저 자동화, 큰 컨텍스트 |

시스템 여유분은 직접 조정할 수 있습니다. 모든 계산은 브라우저 안에서만 실행됩니다.

## 한계

이 결과는 벤치마크가 아니라 계획을 위한 추정치입니다. 실제 동시 실행 수는 CPU 코어 수, 모델 제공자의 동시 요청 제한, 프로세스 오버헤드, 저장소 크기, 작업 간 의존성 때문에 더 낮아질 수 있습니다.

브라우저 RAM 감지는 사용자가 버튼을 눌렀을 때만 실행되며 대략적인 값입니다. 일부 브라우저는 이 정보를 제공하지 않고, Chromium 계열 브라우저는 의도적으로 정밀도를 낮출 수 있습니다. 직접 입력한 값을 기준으로 사용하세요.

## 로컬 실행

패키지 설치나 빌드 과정이 필요하지 않습니다.

```bash
python3 -m http.server 4173
```

브라우저에서 `http://127.0.0.1:4173/`을 여세요.

## 테스트

계산 모듈은 Node 내장 테스트 러너를 사용합니다.

```bash
node --test tests/calculator.test.mjs
```

## 주요 파일

- `calculator.mjs`: 계산과 입력 검증을 담당하는 순수 로직입니다.
- `app.js`: 브라우저 상호작용과 결과 렌더링을 담당합니다.
- `index.html`과 CSS 파일: 접근성을 고려한 정적 인터페이스입니다.
- `showcase.html`: 컴포넌트와 상태 쇼케이스입니다.
- `tests/calculator.test.mjs`: 경계값과 계산식 테스트입니다.
