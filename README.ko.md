# AgentFit

[English](README.md) · [계산기 열기](https://tygb99.github.io/ram-subagent-calculator/)

AgentFit은 안전한 동시 실행 수와 실측 기반 극한 누적 서브에이전트 수를 분리해서 보여주는, 설치가 필요 없는 공개 웹 계산기입니다.

전체 RAM을 입력하고 작업 강도를 선택하면 서로 다른 두 용량 모델을 비교할 수 있습니다.

- **안전 동시 실행**: 스왑에 의존하지 않고 메모리를 사용하는 작업을 동시에 돌릴 수 있는 수입니다.
- **극한 누적 생성**: 스왑과 응답 지연을 감수하며 시간에 걸쳐 생성할 수 있는 논리 서브에이전트 스레드 수입니다.

안전 동시 실행 모델은 다음 세 단계를 함께 보여줍니다.

- **안전**: 계산된 최대치의 65%로, 장시간 작업에 적합한 보수적 수치입니다.
- **균형**: 최대치의 85%로, 기본 권장값입니다.
- **최대**: 메모리만 고려한 계산상 상한이며 갑작스러운 메모리 증가에 취약합니다.

## 계산 방식

운영체제와 다른 앱이 쓸 메모리를 먼저 남기고, 메인 에이전트 1개에 필요한 RAM을 제외한 뒤 남은 풀을 서브에이전트에 배분합니다.

```text
시스템 여유분   = clamp(전체 RAM × 25%, 3 GB, 16 GB)
서브에이전트 풀 = max(0, 전체 RAM − 시스템 여유분 − 메인 에이전트 RAM)
최대            = floor(서브에이전트 풀 ÷ 에이전트당 RAM)
안전            = floor(최대 × 65%)
균형            = floor(최대 × 85%)
```

극한 추정은 한 번의 macOS 실측 결과를 RAM 비율로 환산합니다.

```text
극한 누적 생성 = floor(전체 RAM ÷ 24 GB × 160)
추정 스왑      = 전체 RAM ÷ 24 GB × 21.69 GB
전체 메모리    = 전체 RAM + 추정 스왑
```

실측 환경은 물리 RAM 24GB였으며 스왑 21.69GB, 합산 약 45.7GB까지 사용했습니다. 약 21시간 동안 중복 없는 직계 자식 스레드 160개가 누적됐지만, 메모리를 많이 쓰는 작업 160개를 동시에 실행한 것은 아닙니다. 스레드 메타데이터의 최대 생존 구간 겹침은 24개였고, 후반 작업의 실제 동시성은 3개로 제한했습니다.

계산된 최대값이 1이면 안전과 균형도 1로 유지합니다.

| 작업 강도 | 에이전트당 RAM | 적합한 작업 |
| --- | ---: | --- |
| 가벼움 | 1 GB | 코드 탐색, 작은 수정, 짧은 프롬프트 |
| 균형 | 2 GB | 일반 구현과 리뷰 |
| 무거움 | 4 GB | 빌드, 브라우저 자동화, 큰 컨텍스트 |

시스템 여유분은 직접 조정할 수 있습니다. 모든 계산은 브라우저 안에서만 실행됩니다.

## 한계

안전 수치는 계획을 위한 추정치이며, 극한 수치는 단일 컴퓨터 실측값을 비례 환산한 결과로 보장값이 아닙니다. CPU 코어 수, 모델 제공자의 동시 요청 제한, 프로세스 오버헤드, 저장소 크기, 작업 시간과 의존성에 따라 결과가 달라질 수 있습니다.

극한 누적 수를 동시 실행 권장값으로 사용하지 마세요. 스왑은 심한 지연과 저장장치 부담을 만들 수 있으며, 논리 에이전트 스레드는 빌드·브라우저·추론·미디어 처리처럼 실제로 실행 중인 무거운 작업보다 훨씬 가볍습니다.

브라우저 RAM 감지는 사용자가 버튼을 눌렀을 때만 실행되며 대략적인 값입니다. 일부 브라우저는 이 정보를 제공하지 않고, Chromium 계열 브라우저는 의도적으로 정밀도를 낮출 수 있습니다. 직접 입력한 값을 기준으로 사용하세요.

헤더의 스타 수는 GitHub의 낮은 비인증 브라우저 호출 한도를 피하기 위해 Shields.io의 캐시된 JSON 주소에서 가져옵니다. 캐시 주소가 실패하면 GitHub 공개 API를 사용합니다. 두 주소가 모두 실패해도 계산기는 계속 작동하며 스타 숫자만 표시하지 않습니다.

## 로컬 실행

패키지 설치나 빌드 과정이 필요하지 않습니다.

```bash
python3 -m http.server 4173
```

브라우저에서 `http://127.0.0.1:4173/`을 여세요.

## 테스트

계산 모듈은 Node 내장 테스트 러너를 사용합니다.

```bash
node --test tests/*.test.mjs
```

## 주요 파일

- `calculator.mjs`: 계산과 입력 검증을 담당하는 순수 로직입니다.
- `app.js`: 브라우저 상호작용과 결과 렌더링을 담당합니다.
- `github-stats.mjs`: GitHub 공개 스타 수를 안전하게 가져옵니다.
- `index.html`과 CSS 파일: 접근성을 고려한 정적 인터페이스입니다.
- `showcase.html`: 컴포넌트와 상태 쇼케이스입니다.
- `tests/*.test.mjs`: 계산식, 경계값, GitHub 응답 테스트입니다.
