RAM-AWARE CONCURRENCY

RAM을 입력하면,적정 개수가 보입니다.

시스템과 메인 에이전트가 쓸 메모리를 먼저 남기고, 서브에이전트가 동시에 실행될 수 있는 범위를 계산합니다.

01 / 입력

내 실행 환경

GB

4GB 단위로 입력하세요. 감지는 선택 사항이며 브라우저에 따라 제한될 수 있습니다.

4GB부터 256GB까지, 4GB 단위로 입력해 주세요.

에이전트당 작업 강도

코드 탐색은 가벼움, 일반 구현은 균형, 빌드·브라우저·대형 컨텍스트는 무거움을 권장합니다.

시스템 여유분 조정
GB

자동값은 전체 RAM의 25%이며 최소 3GB, 최대 16GB입니다.

전체 RAM 안에서 0GB 이상의 값을 입력해 주세요.

03 / 기준

숫자를 해석하는 법

A

안전

장시간 실행하거나 다른 앱을 함께 쓸 때 시작하기 좋은 보수적 수치입니다.

B

균형

일반적인 코딩 작업에서 속도와 갑작스러운 메모리 증가를 함께 고려한 권장값입니다.

C

최대

계산상 상한입니다. 스왑과 응답 지연을 피하려면 지속 실행에는 권장하지 않습니다.

이 계산기는 RAM만 봅니다. CPU 코어 수, 모델 제공자의 동시 요청 제한, 작업 간 의존성이 실제 병렬 개수를 더 낮출 수 있습니다.

계산식과 소스는 공개되어 있습니다.
